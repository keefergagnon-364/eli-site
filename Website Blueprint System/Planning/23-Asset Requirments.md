# 23 - Project Readiness & Asset Requirements Prompt

## Purpose

Determine whether the project is ready to enter the development phase.

This prompt serves as the official project gatekeeper.

Do not assume information exists.

Verify every required asset, requirement, and project dependency before allowing development to begin.

The objective is to:

* Identify missing assets
* Prevent development delays
* Generate client action items
* Recommend AI-generated temporary alternatives
* Determine overall project readiness
* Improve website quality

# Project Gatekeeper

This prompt determines whether development should begin.

Development may only proceed when:

✓ Required project information is complete

✓ Critical assets are available

✓ Major risks are identified

✓ The Project Readiness Score meets the minimum confidence threshold defined in [[generalInstructions]]

If readiness is below the threshold:

STOP development.

Generate the missing requirements before continuing.

Never assume missing information.

---

# Readiness Philosophy

Evaluate both:

## Project Readiness

and

## Asset Readiness

A project with complete assets but poor planning is **not** ready.

A project with excellent planning but missing critical assets is **not** ready.

# Failure Conditions

The report fails review if:

* Assets are assumed without verification
* Critical missing items are overlooked
* Ownership is not assigned
* AI alternatives are not considered
* Risks are not identified
* Readiness score is omitted
* Development proceeds below the required confidence threshold

Revise until all failure conditions are resolved.
