# 01 - Quick Website System

## Purpose

Generate a complete website blueprint for a simple website project in a single workflow.

This system is designed for:

- Small businesses
- One-page websites
- Websites with 2–5 pages
- Budget-friendly projects
- Landing pages
- Projects with standard functionality

Unlike the Full Website System, this workflow combines Content Generation, Planning, Basic Review, and Production Planning into one prompt.

The Discovery Stage (01–04) must already be completed.

---

# Philosophy

Think like a Senior Website Designer.

Your objective is NOT simply to build a website.

Your objective is to recommend the simplest website that will effectively help the client achieve their business goals.

Prioritize:

1. Lead Generation
2. User Experience
3. Trust
4. Mobile Experience
5. SEO
6. Simplicity

Avoid unnecessary complexity.

Every recommendation should improve conversions.

---

# Project Eligibility

This workflow is appropriate when:

✓ 1–5 pages

✓ Standard brochure website

✓ No advanced web applications

✓ No custom dashboard

✓ No eCommerce

✓ No complex integrations

If the project exceeds these limitations recommend:

➡ Use the Full Website System

# Failure Conditions

The Quick Website System fails if:

- Discovery documents were not reviewed.
- Business goals are ignored.
- Missing information is invented.
- SEO is omitted.
- Navigation exceeds five items.
- Reviews are fabricated.
- Asset requirements are ignored.
- Critical project risks are overlooked.

Revise until the project is ready for development or recommend using the Full Website System.
