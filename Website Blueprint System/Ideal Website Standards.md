# Ideal Website Standards

## Purpose

This document defines the minimum quality standards for all websites built within the Website Blueprint System.

These standards represent proven best practices for:

* Local Service Businesses
* Contractors
* Landscaping Companies
* Tree Services
* Small Retail Businesses
* Restaurants

If a recommendation conflicts with these standards, the conflict must be documented and justified.

---

# Core Philosophy

Every element on a website must justify its existence.

If an element does not improve one or more of the following:

* Lead Generation
* User Experience
* Trust
* SEO
* Accessibility

it should be removed.

The goal is not to create the flashiest website.

The goal is to create a website that:

* Looks professional
* Builds trust
* Generates leads
* Performs well in search engines
* Works flawlessly on mobile devices

---

# Standard Homepage Structure

Unless project requirements indicate otherwise, websites should follow this structure:

1. Header / Navigation
2. Hero Section
3. Trust Bar
4. Services
5. Why Choose Us
6. About Section
7. Gallery
8. Testimonials
9. FAQ
10. Contact Section
11. Footer

This structure follows a natural customer decision process:

Attention → Trust → Services → Credibility → Proof → Objection Handling → Conversion

---

# Header Standards

Required:

* Business Name or Logo
* Navigation Menu
* Visible Contact Method
* Mobile Navigation

Maximum Navigation Items:

* 5 Primary Navigation Items

Navigation must remain simple and intuitive.

---

# Hero Section Standards

Required:

* Business Name
* Primary Headline
* Supporting Visual
* Primary Call To Action

The Hero Section must clearly communicate:

* Who the business is
* What the business does
* Why the visitor should care

Visitors should understand the business within 5 seconds.

---

# Trust Bar Standards

Include whenever available:

* Years in Business
* Licensed Status
* Insured Status
* Certifications
* Number of Projects
* Customer Rating
* Awards

Trust indicators should appear near the top of the homepage.

---

# Services Standards

Services should focus on customer outcomes.

Avoid:

* Technical jargon
* Feature-heavy descriptions

Focus on:

* Customer problems
* Solutions
* Benefits

Every service should include a call-to-action.

---

# Why Choose Us Standards

Required on all service websites.

Must answer:

* Why choose this company?
* What makes them different?
* Why should customers trust them?

Use specific evidence whenever possible.

---

# About Section Standards

Must humanize the business.

Include when available:

* Company Story
* Team Information
* Experience
* Mission
* Values

Customers prefer doing business with people, not faceless companies.

---

# Gallery Standards

Preferred:

* Real Project Photos
* Before and After Photos
* Team Photos

Avoid excessive stock photography.

Real photography should always be prioritized.

---

# Testimonial Standards

Include whenever available.

Display:

* Reviews
* Testimonials
* Customer Success Stories

Trust-building content should appear before major conversion sections.

---

# FAQ Standards

Required on all websites.

Purpose:

* Answer common questions
* Reduce objections
* Improve SEO
* Improve user experience

Questions should come from real customer concerns whenever possible.

---

# Contact Standards

Required:

* Phone Number
* Contact Form
* Service Area Information

Preferred:

* Email Address
* Business Hours
* Google Business Profile

Contacting the business should be effortless.

---

# Footer Standards

Required:

* Contact Information
* Navigation Links
* Copyright Information

Preferred:

* Service Areas
* Social Links

---

# Content Standards

Content must be:

* Clear
* Professional
* Friendly
* Trustworthy

Avoid:

* Fluff
* Keyword stuffing
* Unnecessary jargon

Write for people first.

Optimize for search engines second.

---

# SEO Standards

Every website should include:

* Meta Title
* Meta Description
* Single H1
* Proper Heading Structure
* Alt Text
* Internal Linking
* Service Area Content
* Local Keywords
* Schema Markup
* Sitemap.xml
* robots.txt

Generate additional SEO recommendations when opportunities exist beyond website implementation.

---

# Mobile Standards

All websites must be mobile-first.

Required:

* Responsive Design
* No Horizontal Scrolling
* Touch-Friendly Buttons
* Readable Typography
* Click-To-Call Phone Number

Mobile usability is a top priority.

---

# Performance Standards

Target Metrics:

* Page Load Time Under 3 Seconds

Lighthouse Scores:

* Performance: 90+
* Accessibility: 95+
* SEO: 95+
* Best Practices: 95+

Optimize:

* Images
* CSS
* JavaScript

Avoid unnecessary dependencies.

---

# Accessibility Standards

Required:

* Semantic HTML
* Alt Text
* Proper Labels
* Visible Focus States
* Sufficient Color Contrast

Accessibility should never be sacrificed for aesthetics.

---

# Technical Standards

Preferred Stack:

* HTML
* CSS
* JavaScript

Use lightweight custom code whenever practical.

Use third-party integrations only when they provide clear business value.

Examples:

* Forms
* Booking Systems
* CRM Integrations
* Jobber Integrations

---

# Conversion Standards

Every homepage must clearly answer:

1. Who are you?
2. What do you do?
3. Who do you serve?
4. Why should I trust you?
5. How do I contact you?

Required CTA Placement:

* Primary CTA in Hero Section
* Final CTA near Bottom of Page

---

# Failure Conditions

A website fails review if any of the following are present:

* Confusing Navigation
* Broken Mobile Experience
* Slow Loading Speeds
* Poor Color Contrast
* Missing CTA
* Missing Contact Information
* Excessive Text Blocks
* Tiny Buttons
* Outdated Design
* Missing SEO Fundamentals

These issues must be resolved before project approval.

---

# Continuous Improvement

This document should be updated after every completed website project.

When new lessons are learned:

* Add new standards
* Refine existing standards
* Remove outdated practices

The goal is to continuously improve the Website Blueprint System over time.
