# Primary Goal

The purpose of the website is to:

1. Generate qualified leads.
2. Generate quote requests.
3. Build brand awareness.
4. Increase phone calls.
5. Support online bookings when applicable.

All recommendations should support one or more of these goals.

---

# Target Client Types

Primary industries:

* Local Service Businesses
* Contractors
* Landscaping Companies
* Tree Services
* Small Retail Businesses
* Restaurants

Recommendations should prioritize best practices that perform well for these industries.

---

# Design Philosophy

Websites should be:

* Clean
* Modern
* Friendly
* Professional
* Trustworthy
* Easy to navigate

Avoid:

* Cluttered layouts
* Tiny buttons
* Excessive animations
* Poor contrast
* Overly complicated navigation

Users should understand the business and next action within seconds of landing on the website.

---

# User Experience Standards

Prioritize:

1. Mobile usability
2. Clear navigation
3. Fast loading speeds
4. Readable content
5. Strong calls-to-action

All layouts should be mobile-first.

Large touch-friendly buttons should be used throughout the website.

---

# Content Standards

Content should be:

* Clear
* Professional
* Friendly
* Trustworthy
* Easy to understand

Avoid unnecessary marketing jargon.

Write for real customers rather than search engines.

Content should focus on:

* Benefits
* Trust
* Clarity
* Problem solving

Do not add filler content simply to increase word count.

---

# SEO Standards

SEO is a primary objective.

All recommendations should prioritize:

* Local SEO
* Service-based SEO
* Search intent
* Proper heading hierarchy
* Metadata optimization
* Internal linking opportunities

When applicable include:

* Service area sections
* Location-based keywords
* Local search intent phrases

Google Business Profile integration is recommended when available.

---

# Accessibility Standards

Accessibility is a high priority.

All websites should:

* Use semantic HTML
* Maintain strong color contrast
* Support keyboard navigation
* Include image alt text
* Use proper heading structure
* Use descriptive links and buttons

Accessibility should never be sacrificed for visual design.

---

# Technical Standards

Preferred stack:

* HTML
* CSS
* JavaScript

Prefer lightweight custom code.

Avoid unnecessary frameworks and dependencies.

Plugins may be used when they provide clear business value such as:

* Forms
* Booking systems
* CRM integrations
* Jobber integrations

The website should remain easy to maintain and update.

---

# Trust Building Standards

Trust elements should be included whenever available.

Examples:

* Reviews
* Testimonials
* Experience
* Certifications
* Licenses
* Guarantees
* Team information
* Project galleries

Trust-building content should appear before major conversion points whenever possible.

---

# Missing Information Policy

Never make assumptions.

If required information is missing:

1. Identify the missing information.
2. Generate a Missing Information Report.
3. Explain why the information is needed.
4. Request clarification.

Do not fabricate business information.

Do not invent statistics, reviews, credentials, certifications, awards, or guarantees.

---

# Confidence Requirement

Before Final Website Assembly begins:

Minimum Confidence Score: 95%

If confidence is below 95%:

* Generate a Missing Information Report.
* Identify unanswered questions.
* Request clarification.
* Delay assembly until confidence threshold is met.

---

# Decision Hierarchy

When conflicts arise:

1. Project Scope
2. Website Strategy
3. Ideal Website Standards
4. General Instructions
5. Individual Prompt Rules

Higher-priority documents override lower-priority documents.

---

# Success Criteria

A successful website should:

* Look professional
* Stand out from competitors
* Be easy to navigate
* Load quickly
* Perform well on mobile devices
* Build trust
* Generate leads
* Support local SEO
* Be easy for the client to maintain

Every recommendation should move the project closer to these outcomes.
