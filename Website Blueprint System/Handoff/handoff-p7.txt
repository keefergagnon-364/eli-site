Production Handbook
Part 7 — Asset Management, Media Standards & Placeholder Library

Document Type: Asset Management Specification

Status: Design Freeze Approved

Authority: This document defines every media asset, placeholder policy, naming convention, optimization requirement, replacement workflow, and asset management standard used throughout the IVY Remodeling website.

Purpose

This document ensures that all visual assets are:

Consistent
Optimized
Easy to replace
Organized
Scalable
SEO friendly

The implementation team shall treat media assets as part of the architecture—not decorative content.

Asset Philosophy

Every image should accomplish at least one objective:

Build trust
Explain a service
Showcase craftsmanship
Improve usability
Support SEO

No image should exist solely to fill empty space.

Naming Convention

Use descriptive, lowercase, hyphenated filenames.

Examples:

hero-deck-construction.webp

eli-doyle-placeholder.webp

deck-before-01.webp

deck-after-01.webp

trex-deck-project-02.webp

Avoid:

IMG_1023.jpg

photo1.png

image-final-final.jpg

new-image.webp

Image Optimization Standards

Every image should be optimized before deployment.

Recommended maximum sizes:

Hero

250–400 KB

Gallery

100–250 KB

Service Images

100–200 KB

Icons

<10 KB

Favicons

As small as practical while retaining clarity.

Responsive Images

Every major photograph should provide multiple sizes.

Example:

hero-480.webp

hero-768.webp

hero-1200.webp

hero-1920.webp

Use <picture> or srcset where appropriate.

Hero Image

Purpose:

Immediately communicate professionalism and craftsmanship.

Requirements:

High quality
Natural lighting
Arizona-style residential setting
Large composition
Space for text overlay
Fast loading

Avoid cluttered compositions that compete with headlines.

Service Images

Every service page requires:

Hero image

Supporting image(s)

Detail photography

Optional before/after comparison

All images should accurately represent the service.

Gallery Images

Gallery images should emphasize:

Finished work
Quality details
Materials
Craftsmanship
Variety of projects

Avoid repetitive angles or nearly identical images.

Before / After Images

Every pair should follow consistent standards:

Same angle

Similar lighting where possible

Comparable framing

Clear visual improvement

Each pair should include descriptive captions.

Placeholder Asset Policy

Approved placeholders include:

Owner portrait
Additional project photography
Before/after examples
Gallery images
Review profile photos (if used)

Requirements:

High-quality
Commercially licensed
Similar style to final imagery
Easy replacement
Clearly documented in the asset inventory
Placeholder Replacement Workflow

When a real asset becomes available:

Replace the placeholder file.
Keep the same filename if practical.
Preserve dimensions.
Verify responsive versions.
Confirm alt text.
Re-test layout.

The replacement process should never require HTML or CSS changes.

Logo Standards

Primary format:

SVG

Fallback:

PNG

Requirements:

Transparent background

Sharp rendering

Consistent spacing

Adequate clear space

Never stretch or distort the logo.

Social Sharing Images

Every major page should have an Open Graph image.

Recommended size:

1200 × 630 px

Include:

Logo
Brand colors
Minimal text
Project photography where appropriate
Alt Text Standards

Every image must include descriptive alt text.

Good example:

Newly constructed Trex deck overlooking a wooded backyard in Show Low, Arizona.

Poor example:

Deck

Decorative graphics should use empty alt attributes.

Lazy Loading Policy

Lazy load:

Gallery images
Below-the-fold photography
Background images where practical

Do not lazy load:

Hero image
Logo
Above-the-fold trust icons
Image SEO

Every image should include:

Descriptive filename
Alt text
Width/height attributes
Responsive sizes
Appropriate compression

Images should contribute to local SEO where relevant.

Media Accessibility

Ensure:

Meaningful alt text
No text embedded in critical images
Sufficient contrast for overlays
Captions where helpful
Accessible lightbox behavior
