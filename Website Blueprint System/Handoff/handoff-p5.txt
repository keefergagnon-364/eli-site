Production Handbook
Part 5 — SEO, Technical Architecture & Development Standards

Document Type: Engineering & Implementation Specification

Status: Design Freeze Approved

Authority: This document defines the technical standards, SEO implementation, coding architecture, performance requirements, and development rules for the IVY Remodeling website.

Purpose

The purpose of this document is to define how the website must be built, not how it should look.

The production team shall follow these standards to ensure:

Maintainability
Performance
Accessibility
SEO
Scalability
Code consistency

This website is intended to be production-quality, not a prototype.

Development Philosophy

The website should prioritize:

Maintainability
Performance
Accessibility
SEO
Progressive Enhancement
Scalability

Fancy effects should never come at the expense of usability or performance.

Technology Stack

Approved technologies:

HTML

HTML5

Semantic HTML only.

Avoid unnecessary wrapper elements.

Every section should use the appropriate semantic element.

Examples:

<header>
<main>
<section>
<article>
<aside>
<footer>
<nav>
CSS

Vanilla CSS

No CSS frameworks.

No Bootstrap.

No Tailwind.

No UI libraries.

Reason:

Maintain full control over performance, styling, and future maintenance.

JavaScript

Vanilla JavaScript

Use ES6 modules where appropriate.

Avoid unnecessary dependencies.

Every script should have a clear responsibility.
