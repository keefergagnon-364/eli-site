Production Handbook
Part 4 — Visual Design System & Component Library

Document Type: Visual Implementation Specification

Status: Design Freeze Approved

Authority: This document defines the visual identity, UI system, reusable components, responsive behavior, animation standards, and accessibility rules for the IVY Remodeling website.

Purpose

The purpose of this document is to eliminate subjective design decisions during implementation.

The development team is responsible for implementing the approved visual system—not designing a new one.

Every page should feel like it belongs to the same website.

Every component should be reusable.

Consistency is more important than creativity.

Overall Design Philosophy

The website should communicate one impression above everything else:

Professional craftsmanship with an approachable, local personality.

Visitors should immediately feel that IVY Remodeling is:

Experienced
Trustworthy
Detail-oriented
Skilled
Honest
Premium without being luxurious

The design should never feel:

Corporate
Cold
Flashy
Over-designed
Cheap
Template-based
Visual Personality

If the design were a physical space, it would resemble:

A well-built custom cabin
A professionally crafted workshop
A modern mountain home
Natural wood textures
Clean architectural lines
Warm lighting
Organized tools
Quality materials

Every design decision should reinforce this feeling.

Ivory should provide breathing room.

Wood Brown should appear sparingly.

Bright colors should never distract from content.

The website should remain calm and timeless.

Typography Rules

Never center long paragraphs.

Avoid large walls of text.

Maintain generous line height.

Keep paragraph widths readable.

Avoid decorative fonts.

Spacing System

Whitespace is intentional.

It creates professionalism.

Adopt a consistent spacing scale (e.g., 8px base unit) so margins, padding, and component gaps remain proportional throughout the site.

Pages should breathe.

Avoid crowded layouts.

Layout System

Maximum content width:

Approximately 1200–1400px.

Sections should alternate subtly using:

Ivory
White
Light neutral backgrounds

Large imagery should break up text.

Animations

Animations should support usability.

Allowed:

Fade

Slide

Hover elevation

Button feedback

Accordion transitions

Scroll reveal

Not allowed:

Autoplay motion

Parallax

Flashing effects

Large entrance animations

Animation should never delay content visibility.

Responsive Design

Breakpoints should support:

Desktop

Laptop

Tablet

Large phones

Small phones

Layout should adapt naturally without creating horizontal scrolling.

Navigation should collapse gracefully.

Cards should stack cleanly.

Forms should remain easy to complete.

Accessibility Standards

Minimum target:

WCAG 2.2 AA

Requirements:

Semantic HTML

High color contrast

Visible focus indicators

Keyboard navigation

Alt text

Accessible labels

Descriptive buttons

Touch targets ≥44×44px

No color-only communication

Performance Budget

Visual quality must never compromise performance.

Targets:

Optimized WebP images (with fallbacks if needed)
Lazy loading
SVG icons
Minimal font weights
Limited animations
No autoplay video
Efficient CSS architecture
Minimal JavaScript

Performance is a design requirement.
